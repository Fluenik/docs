---
title: Connect to the gateway
description: "Issue an API key in your dashboard and point any MCP-compatible client at the BRAM Gateway endpoint. No install required."
---

BRAM Gateway is hosted — there's nothing to install. Your agents authenticate with an API key and connect to your workspace's gateway URL. Both the key and the URL are issued from [app.getbram.com](https://app.getbram.com).

## 1. Issue an API key

<Steps>
  <Step title="Sign in">
    Sign in at [app.getbram.com](https://app.getbram.com).
  </Step>
  <Step title="Create a workspace">
    Every new account gets a workspace. If you're working with a team, you can invite them later from **Settings → My Team**.
  </Step>
  <Step title="Open API Keys">
    In the left nav, click **API Keys → New key**. Name the key after the agent that will use it (e.g. `support-bot-prod`).
  </Step>
  <Step title="Copy the key + endpoint">
    BRAM shows you the plaintext API key **once** along with the gateway endpoint URL. Copy both — the key can't be retrieved again, only revoked and reissued.

    ```bash
    export BRAM_API_KEY="brk_live_7f2a..."
    export BRAM_GATEWAY_URL="https://gateway.getbram.com/<workspace-slug>/mcp"
    ```
  </Step>
</Steps>

<Warning>
  The plaintext API key is shown **once at creation** and can't be retrieved again. If you lose it, revoke it from the dashboard and issue a new one.
</Warning>

## 2. Register your upstream MCP servers

BRAM doesn't replace your MCP servers — it proxies them. Register each upstream server in the dashboard so the gateway knows where to forward calls.

In the left nav, open **MCP Servers → Add server**. Provide:

- **Server URL** (for HTTP-transport MCP servers) or **command + args** (for stdio servers BRAM should manage)
- **Name** the agent will see this prefix as (e.g. `github`, `filesystem`, `slack`)
- **Auth** — bearer token, header, or none, depending on the server

When you save, BRAM hot-loads the server and exposes the union of all registered servers' tools to every connected agent, namespaced by name (`github.create_issue`, `filesystem.read_file`, etc.).

## 3. Verify the gateway is reachable

```bash
curl -fsS "$BRAM_GATEWAY_URL/health" \
  -H "Authorization: Bearer $BRAM_API_KEY"
```

Expected output:

```json
{
  "status": "ok",
  "workspace": "your-workspace-slug",
  "mcp_servers": 3,
  "policy_version": 7
}
```
