---
title: Connect your agent
description: "Point Claude Desktop, Cursor, Cline, or any MCP-compatible client at your BRAM Gateway endpoint."
---

BRAM Gateway exposes a standard MCP HTTP endpoint at the URL shown alongside your API key in [app.getbram.com](https://app.getbram.com) — typically `https://gateway.getbram.com/<workspace-slug>/mcp`.

The pattern is the same for every client: **replace the MCP server URL in your client config with the BRAM Gateway URL**, and add the API key as a bearer token. Upstream MCP servers are registered once in the BRAM dashboard, not per-client.

<Note>
  Throughout this page we'll use `$BRAM_GATEWAY_URL` and `$BRAM_API_KEY` as placeholders. You can find the real values in **API Keys → your key** in the BRAM dashboard.
</Note>

## Claude Desktop

Edit your Claude config file:

- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

Replace any `mcpServers` entries with a single BRAM entry:

```json
{
  "mcpServers": {
    "bram": {
      "url": "https://gateway.getbram.com/<workspace-slug>/mcp",
      "headers": {
        "Authorization": "Bearer brk_live_..."
      }
    }
  }
}
```

Restart Claude Desktop. Claude now talks to BRAM Gateway; BRAM forwards each call to the upstream MCP servers you've registered in the dashboard.

## Cursor

Open **Cursor Settings → MCP → + Add new MCP Server**:

```json
{
  "mcpServers": {
    "bram": {
      "url": "https://gateway.getbram.com/<workspace-slug>/mcp",
      "headers": {
        "Authorization": "Bearer brk_live_..."
      }
    }
  }
}
```

Save. Cursor's composer will now route every tool call through BRAM.

## Custom MCP clients

The endpoint implements the standard [MCP HTTP transport](https://modelcontextprotocol.io/docs/concepts/transports). Any compliant client works:

```ts
import { Client } from '@modelcontextprotocol/sdk/client/index.js'
import { StreamableHTTPClientTransport } from '@modelcontextprotocol/sdk/client/streamableHttp.js'

const transport = new StreamableHTTPClientTransport(
  new URL(process.env.BRAM_GATEWAY_URL!),
  {
    requestInit: {
      headers: {
        Authorization: `Bearer ${process.env.BRAM_API_KEY!}`,
      },
    },
  },
)

const client = new Client(
  { name: 'my-agent', version: '1.0.0' },
  { capabilities: {} },
)
await client.connect(transport)
```

## What your agent sees

The gateway exposes the **union** of every registered upstream server's tools, namespaced by server name:

```
filesystem.read_file
filesystem.write_file
github.create_issue
github.search_repositories
slack.post_message
```

Each tool call goes through the policy engine and inline guardrails before being forwarded upstream. The response — success, error, or policy denial — is streamed back as a standard MCP tool result, with the matched policy/version/decision visible in the trace ID returned with every response.

## Per-agent identity

The API key authenticates the **agent**, not the human. Issue a separate key per agent so:

- Decision logs show which agent did what
- Per-agent kill switch lets you revoke one agent's access without touching the others
- Per-agent usage budgets cap tokens, runs, or cost per agent
